# 0.5 Compatibility Removal

## Purpose

This note records which compatibility seams were removed, which remain bounded, and why they do not count as surviving legacy authority.

## Removal Closure

### Removed as live authority

- legacy Todoist task mutation routes now return explicit deprecation errors instead of calling legacy writeback services
- non-`0.5` provider write routes for Notes, Reminders, GitHub, and Email now return explicit deprecation errors instead of silently surviving the cutover

### Bounded compatibility that remains

- integration settings/status read surfaces
- connection/log/event read surfaces
- local source-path configuration surfaces
- bounded sync/read surfaces required for operator continuity during backend-only `0.5`

These remaining paths are bounded because they do not own canonical write authority.

## Removal Criteria

Compatibility scaffolding is considered closed when all of the following are true:

1. No legacy route performs provider mutation.
2. Todoist and Google write execution terminate on canonical `WriteIntent` routes.
3. Remaining compatibility surfaces are read-only or configuration-only.
4. Any non-`0.5` provider write path fails closed with explicit deferred posture.

## Quarantine Rule

Read compatibility may remain temporarily quarantined.

Write compatibility may not.

If a legacy path still performs writes, Phase 65 fails.
