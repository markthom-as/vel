# 0.5 Cutover Notes

## Live Canonical Backend Authority

The `0.5` cutover makes the canonical backend the only live write authority for the proving adapters:

- `/api/integrations/todoist/write-intent`
- `/api/integrations/google-calendar/write-intent`

These routes terminate on the `0.5` bridge stack:

- policy evaluation
- conflict classification
- canonical `WriteIntent` creation/dispatch
- explain payload generation
- audit emission

No legacy Todoist or Google write route remains authoritative after cutover.

## Retired Legacy Write Paths

The following legacy writeback routes are retired and now fail closed with explicit deprecation posture:

- `/api/integrations/todoist/create-task`
- `/api/integrations/todoist/update-task`
- `/api/integrations/todoist/complete-task`
- `/api/integrations/todoist/reopen-task`
- `/api/integrations/notes/create-note`
- `/api/integrations/notes/append-note`
- `/api/integrations/reminders/create`
- `/api/integrations/reminders/update`
- `/api/integrations/reminders/complete`
- `/api/integrations/github/create-issue`
- `/api/integrations/github/add-comment`
- `/api/integrations/github/close-issue`
- `/api/integrations/github/reopen-issue`
- `/api/integrations/email/create-draft-reply`
- `/api/integrations/email/send-draft`

These routes are no longer shadow authorities. They exist only as explicit deprecated cutover failures.

## Isolated Read And Configuration Surfaces

The following surfaces remain available during `0.5` as isolated read/configuration compatibility seams:

- `/api/integrations`
- `/api/integrations/connections`
- `/api/integrations/connections/:id`
- `/api/integrations/connections/:id/events`
- `/api/integrations/:id/logs`
- `/api/integrations/:id/source`
- `/api/integrations/:id/path-dialog`
- `/api/integrations/google-calendar`
- `/api/integrations/google-calendar/disconnect`
- `/api/integrations/google-calendar/auth/start`
- `/api/integrations/todoist`
- `/api/integrations/todoist/disconnect`
- `/v1/sync/todoist`

These surfaces are tolerated for operator continuity, but they are not live write authority for `0.5`.

## Cutover Rule

Activation and configuration remain distinct from invocation and write execution.

The canonical rule after cutover is:

- provider settings/auth may remain configurable
- provider reads/imports may remain available
- provider writes must go only through canonical `WriteIntent` routes

Any future provider write surface must join the canonical `WriteIntent` path or remain out of scope.
