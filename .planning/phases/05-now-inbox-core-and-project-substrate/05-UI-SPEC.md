## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, inline metadata spacing |
| sm | 8px | Compact row spacing, stacked metadata, small button gaps |
| md | 16px | Default card padding, list row spacing, body rhythm |
| lg | 24px | Section padding, grouped panel spacing |
| xl | 32px | Shell gutters, panel-to-panel separation |
| 2xl | 48px | Major section breaks, empty-state breathing room |
| 3xl | 64px | Page-level spacing for large desktop sections only |

Exceptions: 44px minimum touch target for icon-only or tap-first controls in guided linking, Inbox row actions, and Apple continuity actions carried by this phase.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 | 1.5 |
| Label | 12px | 600 | 1.3 |
| Heading | 20px | 600 | 1.2 |
| Display | 30px | 600 | 1.2 |

Use body text for operational copy, labels for timestamps/status/meta, heading for panel titles, and display only for top-level surface titles such as `Now`, `Inbox`, and guided setup headers.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #09090b | App background, main canvas, full-height shell |
| Secondary (30%) | #18181b | Cards, sidebar, nav rail, grouped panels, placeholder surfaces |
| Accent (10%) | #10b981 | Primary CTA, active shell selection indicator, selected action/intervention row, pairing/link status CTA |
| Destructive | #ef4444 | Destructive actions only |

Accent reserved for: `Create project`, active nav state, selected action/intervention row, link-device CTA, and the single confirm/continue action in guided linking or project setup. Do not spend accent color on every button, link, or badge.

Warning and degraded-state surfaces continue to use the existing amber treatment already established in `Now` and `Settings`. Success feedback may continue to use emerald confirmation text, but only the reserved elements above should take full accent emphasis.

| Element | Copy |
|---------|------|
| Primary CTA | Create project |
| Empty state heading | Inbox is clear |
| Empty state body | No actions need triage right now. Return to `Now` for the current decision set, or open `Threads` to inspect the evidence behind recent state. |

| Destructive confirmation | Unlink device: Remove this device from shared continuity? It will stop syncing until paired again. Dismiss intervention: Dismiss this item from Inbox? ## Surface Hierarchy

- `Now` is the default landing surface and primary decision cockpit.
- Inside `Now`, the primary visual focal point is the ranked action/intervention stack at the top of the main work area, before supporting schedule and source-detail panels.
- `Inbox` is a triage queue for unresolved interventions and newly surfaced work, not a second message list.
- `Projects` is secondary support for work anchoring and may remain a truthful placeholder until the backend project workspace contract exists.
- Daily and weekly loops should route the operator back through `Now` and `Inbox`, not toward a dashboard-first workflow.

---

## Interaction Contract

- Keep degraded or stale information visible with explicit warnings and recovery actions. Do not hide stale schedule, task, or context sections just because freshness degraded.
- Every surfaced action or intervention must show enough metadata to explain why it is here now: kind, current state, surfaced timestamp, and a visible path to evidence or thread context.
- `Inbox` rows should be scannable first and expandable second. Default density favors fast triage over full-detail cards.
- Guided linking must be linear and explicit: issue pairing token, disclose read/write/execute scope, confirm pairing, then show linked status plus fallback path through CLI or web.
- New-project setup stays local-first until operator confirmation. Any upstream creation or linking side effect happens only after an explicit confirm step.
- Desktop preserves the existing shell structure: left navigation, main work area, optional explain/inspection panel. Narrow layouts may collapse to one pane at a time, but the view hierarchy stays the same.
- Icon-only controls are allowed only when they also provide an `aria-label` and desktop tooltip, and they must switch to a visible text label on narrow or touch-first layouts.

---

## Component Inventory

- Reuse the existing three-pane shell, left nav, and panel framing already present in `AppShell`, `Sidebar`, and `MainPanel`.
- Reuse bordered dark cards and state panels as the base visual primitive.
- Use status pills for freshness, link state, and triage state.
- Use warning banners for degraded freshness or trust-reducing conditions.
- Use compact list rows for Inbox/action queues; reserve larger cards for detail views or grouped summaries.
- Keep placeholder surfaces truthful when backend contracts do not exist yet.
