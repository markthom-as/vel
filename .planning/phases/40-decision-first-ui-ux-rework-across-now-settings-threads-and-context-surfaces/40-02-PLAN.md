# 40-02 Plan

## Slice

Publish canonical Rust-owned MVP contracts and typed data models for the full loop.

## Work

- identify the canonical domain/read-model contracts for:
  - overview
  - commitments
  - reflow
  - thread escalation and continuation
  - review and closure
- reconcile those contracts against existing `now`, `daily_loop`, `reflow`, thread, and DTO seams
- specify user-visible fields, state transitions, provenance expectations, and degraded states
- define where durable contracts live in `docs/` and what examples/templates belong near code once boundaries are stable

## Verification

- every MVP loop step maps to a Rust-owned contract or read-model seam
- contracts describe state transitions and degraded behavior, not just shape
- overlaps or conflicts with existing service/DTO seams are called out and resolved

## Exit

Later implementation phases have a concrete contract target instead of inferring product behavior from UI or route code.
