# Phase 101 Plan

## Objective

Freeze the hybrid duplex architecture so later phases cannot blur native-platform policy and portable Rust engine ownership.

## Deliverables

- [ ] locked trait catalog for audio input/output, STT, TTS, and conversation model
- [ ] locked ownership table for native shell vs Rust engine
- [ ] locked real-time threading rules
- [ ] locked latency and validation gates
- [ ] locked proving-adapter requirement before iOS completion claims

## Execution Checklist

1. Write the ownership matrix in milestone docs and phase artifacts.
2. Define the callback-path invariants and banned operations.
3. Define the typed adapter events that cross into Rust.
4. Define the single-active-turn state model and interrupt/cancel semantics.
5. Define the proof obligations for desktop/harness and real iOS device testing.

## Exit Criteria

- implementation phases can proceed without reopening ownership disputes
- validation and verification expectations are written before code spreads
- the phase leaves no ambiguity about whether Rust or native shell owns any privileged Apple audio concern
