# 28-02 Plan

## Slice

Implement backend-owned initial day-plan shaping with explicit scheduled, deferred, and did-not-fit outcomes.

## Why this slice second

Once the contract exists, the next highest-value move is making day shaping real in the backend so morning and daytime flows can consume explainable planning output.

## Outcomes

- compute a bounded same-day plan from calendar anchors, open commitments, routine blocks, and canonical scheduler rules
- emit explicit planned outcomes instead of implicit or shell-local placement logic
- preserve conservative behavior around fixed anchors and judgment-required conflicts
- keep the implementation aligned with the existing `reflow` substrate instead of creating a second planner core

## Likely Touch Points

- `crates/veld/src/services/`
- `crates/vel-storage/src/`
- `crates/vel-core/src/`
- daily-loop and `Now` backend read-model seams

## Verification

- focused backend tests for same-day planning outcomes
- smoke tests over planner outputs for scheduled/deferred/did-not-fit cases
- narrow regression checks that `reflow` still composes cleanly with the new planning seam
