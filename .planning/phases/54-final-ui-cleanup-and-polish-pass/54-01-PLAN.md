# Phase 54 Plan 01

## Goal

Implement the bounded polish set approved in Phase 53 so the corrected surfaces feel intentional, compact, and operator-trusted rather than merely structurally conformant.

## Scope

- refine top nav branding, status cluster, info affordance, and page-title metadata
- fix right-sidebar behavior and contextual documentation treatment
- polish `Now` nudge hierarchy, task presentation, and floating composer behavior
- improve `Threads` filtering, search affordance, row metadata, and sidebar summary
- restore minimum functional settings controls under the refined category map

## Workstreams

### 1. Navigation And Sidebar Polish

- restyle the top nav so `Vel` is unboxed, accented, and visually primary
- move current time/date/current task summary into the nav
- add color-coded nav status icons/counts for `nudges`, `unread threads`, and `sync`
- reduce nav links to icon + smaller text with underline active state
- convert `Documentation` into the top-level info affordance
- correct right-sidebar collapse behavior and viewport sizing
- add contextual documentation placeholder content keyed to the active view

### 2. Now Surface Polish

- remove duplicated time/context from the `Now` top area
- promote backup trust warning into its own nudge lane treatment
- add urgency-weighted nudge styling: smaller actions, better icons, timestamps, project tags, and a highlighted top-priority nudge
- restyle tasks with a subordinate `Tasks` header, active-task emphasis, richer metadata, project color treatment, and right-side summary stats
- add inline task reschedule controls with `Tomorrow`, `Later This Week`, `Pick Day`, and `Unschedule`

### 3. Composer And Threads Polish

- convert the floating composer into a tighter pill-like interaction model with brand emphasis and conditional send affordance
- keep the composer centered relative to the `Now` content lane when the info sidebar opens
- align the `Threads` header scale to `Now`
- add thread filters, integrated search icon, date/last-message metadata, and bottom-sidebar summary text

### 4. Settings Functional Recovery

- keep the compact layout but restore minimum functional settings behavior from the prior version
- restructure the rail to:
  - `Profile`
  - `Clients & Sync`
  - `Integrations`
  - `Configuration`
  - `Backups`
  - `Permissions & Policies`
  - `Templates`
  - `Projects`
- make the compact rail icon-led with smaller centered labels and an icon-only back control
- preserve autosave behavior for the restored editable controls

## Exit Criteria

- accepted Phase 53 review findings are implemented across the affected surfaces
- no new product lanes are introduced
- any remaining rough edges are explicitly documented as non-blocking follow-up rather than silently carried forward
