    - Newtype: `pub struct NodeIdentity(String)` with `fn new(id: impl Into<String>) -> Self` and `Display`, `Serialize`, `Deserialize`

    HlcTimestamp (in vel-core/src/sync.rs):
    - Fields: `wall_time: i64` (unix ms), `logical: u32`, `node_id: NodeIdentity`
    - Ordering: compare wall_time first; if equal compare logical; if equal compare node_id string (lexicographic)
    - `impl Ord + PartialOrd` with deterministic comparator
    - `fn now(node_id: NodeIdentity) -> Self` — creates timestamp at current wall time with logical=0
    - `fn advance(&self, other: &HlcTimestamp, node_id: NodeIdentity) -> HlcTimestamp` — standard HLC advance: wall = max(self.wall, other.wall, now); if wall unchanged increment logical

    Tests (TDD):
    - Test: HlcTimestamp::now(node_a) < HlcTimestamp::now(node_b) when node_b has higher wall_time
    - Test: equal wall_time → higher logical wins
    - Test: equal wall_time AND logical → lexicographically higher node_id wins
    - Test: advance() never goes backward (returned timestamp >= both inputs)
    - Test: same input twice → same output (determinism)
    - Test: conflict resolution: given two events with HlcTimestamps t1 < t2, merge picks t2's value

    sync_ordering_metadata table:
    - Columns: `entity_type TEXT NOT NULL`, `entity_id TEXT NOT NULL`, `hlc_wall_time INTEGER NOT NULL`, `hlc_logical INTEGER NOT NULL`, `hlc_node_id TEXT NOT NULL`, `updated_at INTEGER NOT NULL`
    - Primary key: `(entity_type, entity_id)`

    Reconciliation in client_sync.rs:
    - Add `pub async fn reconcile_with_ordering(storage: &Storage, entity_type: &str, entity_id: &str, incoming_hlc: &HlcTimestamp, local_hlc: Option<&HlcTimestamp>) -> Result<bool, AppError>`
    - Returns `true` if incoming wins (should apply), `false` if local wins (discard)
    - Uses HlcTimestamp::Ord comparison: incoming > local → apply and update metadata; incoming <= local → discard
    - Persists the winning timestamp in sync_ordering_metadata
    - Conflict decision emitted as a log event (traceability per ticket 005)

        ```rust
        pub async fn reconcile_with_ordering(
            storage: &Storage,
            entity_type: &str,
            entity_id: &str,
            incoming: &vel_core::HlcTimestamp,
            node_id: &str,
        ) -> Result<bool, crate::errors::AppError> {
            let local = storage.get_sync_ordering(entity_type, entity_id).await
                .map_err(AppError::from)?;
            let should_apply = match local {
                None => true,  // no existing ordering — always apply
                Some(rec) => {
                    let local_hlc = vel_core::HlcTimestamp {
                        wall_time: rec.hlc_wall_time,
                        logical: rec.hlc_logical as u32,
                        node_id: vel_core::NodeIdentity(rec.hlc_node_id),
                    };
                    incoming > &local_hlc
                }
            };
            if should_apply {
                let now = time::OffsetDateTime::now_utc().unix_timestamp();
                storage.upsert_sync_ordering(
                    entity_type, entity_id,
                    incoming.wall_time, incoming.logical as i64,
                    &incoming.node_id.0, now,
                ).await.map_err(AppError::from)?;
            }
            // Emit SYNC_CONFLICT_DECISION log
            tracing::info!(entity_type, entity_id, applied = should_apply, "sync conflict decision");
            Ok(should_apply)
        }
        ```
