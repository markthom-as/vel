  <behavior>
    Trait contract (in vel-core/src/signal_reducer.rs):
    - `pub trait SignalReducer: Send + Sync` with method `fn name(&self) -> &'static str` and a reduce method
    - Because vel-core cannot import vel-storage, the reduce method takes `serde_json::Value` signal payloads: `fn reduce(&self, ctx: CurrentContextV1, signals: &[SignalPayload]) -> CurrentContextV1`
    - OR: define the trait in veld/services/inference/ (in the service layer) where vel-storage is available — this is cleaner given the layering constraint. The trait then uses `vel_storage::SignalRecord` directly.
    - Decision: define `SignalReducer` trait in `crates/veld/src/services/inference/mod.rs` (service layer, not vel-core) to avoid the layering constraint. Export it from inference module.

    ReducerRegistry behavior:
    - `ReducerRegistry::new()` registers reducers in a fixed, documented order
    - `fn apply_all(&self, ctx: CurrentContextV1, signals: &[SignalRecord]) -> CurrentContextV1` — applies each reducer in order
    - Order must be: [CalendarReducer, MessagesReducer, HealthReducer, GitActivityReducer]
    - Each reducer only touches fields in CurrentContextV1 it owns — other fields pass through unchanged

    CalendarReducer:
    - Test: given empty signals, returns ctx unchanged
    - Test: given signal with `signal_type = "calendar"`, populates ctx.schedule/events fields
    - Test: given two calendar signals, both appear in ctx events

    MessagesReducer:
    - Test: given empty signals, returns ctx unchanged
    - Test: given signal with `signal_type = "message_thread"`, populates ctx.messages fields

    HealthReducer:
    - Test: given signal with `signal_type = "health"`, populates ctx health summary fields
    - Test: given no health signal, ctx health fields remain at default

    GitActivityReducer:
    - Test: given signal with `signal_type = "git_activity"`, populates ctx git summary fields

    Replay determinism:
    - Test: applying ReducerRegistry to the same (ctx, signals) pair twice produces identical output (structural equality)

    inference.rs thin loop:
    - `pub async fn run(storage: &Storage) -> Result<usize, crate::errors::AppError>` — signature unchanged
    - Body: load signals, load commitments, build initial ctx, call `registry.apply_all(ctx, &signals)`, persist result
    - Loop body must be under 50 lines (excluding the registry construction)
  </behavior>
