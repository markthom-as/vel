# Ticket: Add artifacts, logging, and run history

## Objective

Make skill executions operationally legible.

## Deliverables

- run IDs
- persisted execution records
- artifact registry or directory convention
- `vel skill runs`
- `vel skill logs`

## Acceptance criteria

- users can inspect recent runs for a skill
- artifacts are associated with run IDs
- requested/granted capabilities are visible post-run
