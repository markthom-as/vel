Keep ownership of truths across the first three `0.5` implementation phases explicit.

- canonical `content` / `registry` / `read_model` / `runtime` taxonomy

## Phase 58 Established

## Phase 59 Will Establish

- confirmation semantics and precedence rules
- ownership evaluation, stale handling, and conflict classification
- audit/explainability behavior
- `WriteIntent` lifecycle and execution-dispatch contract
- membrane error matrix

## Ownership Rule

- Phase 57 owns ontology and contract law.
- Phase 58 owns substrate and persistence.
- Phase 59 owns lawful action/governance behavior over that substrate.

No later phase should quietly reopen earlier truths without naming the contradiction.
