# Phase 84 Plan

## Objective

Implement the shared shell and token foundation for the `0.5.4` implementation line without widening page-specific behavior.

This phase must land:

- the canonical dark-first graphite/copper token baseline
- the locked typography stack aliases
- the top-band shell chrome
- a docked action bar primitive
- a shell-side nudge zone primitive

## Scope

### In scope

- `clients/web/src/index.css`
- `clients/web/src/core/Theme/tokens.ts`
- `clients/web/src/shell/`
- `clients/web/src/App.tsx`
- targeted shell tests

### Transitional allowance

- `Now` may keep its existing internal nudge rendering until the dedicated `Now` phase if the shell-side zone is established for the shared shell without introducing duplicate top-priority signals.
- the exact Geist / JetBrains assets may remain fallback-backed for this phase because the locked family names are not currently vendored in the repo.

### Out of scope

- full `Now` page rewrite
- thread or system layout rewrite
- row/card taxonomy rebuild
- browser-proof artifact generation

## Implementation slice

1. Replace the old display/token posture with the locked family names and warmer industrial CSS variables.
2. Rework the top shell chrome into a stable top band instead of a thin navbar strip.
3. Add a shared docked action bar with visible, labeled quick actions.
4. Add a shared nudge zone component driven by canonical `Now` state.
5. Keep the surface bodies largely intact so later phases can rebuild them against the new shell.

## Verification

- targeted Vitest coverage for `AppShell`, `Navbar`/top band, and `MainPanel`
- existing route/surface shell tests remain green
- no page-level behavior regressions introduced in `Now`, `Threads`, or `System`
