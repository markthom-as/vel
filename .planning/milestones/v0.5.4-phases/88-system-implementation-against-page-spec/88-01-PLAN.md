# Phase 88 Plan

## Objective

Rebuild `System` into the approved hybrid read-first / operational surface.

## Scope

### In scope

- replace the old domain/capabilities/configuration taxonomy with:
  - `Overview`
  - `Operations`
  - `Integrations`
  - `Control`
  - `Preferences`
- implement browse/detail behavior for `Integrations`
- make `Control` dense but readable with expandable structural rows
- keep provider identity recognizable but subordinate to trust/state
- migrate `System` navigation targets used by `Now` and shell nudges

### Out of scope

- dedicated large-log pages
- deep workflow/schema editors
- backend preference persistence

## Implementation Slice

1. Replace section and subsection routing with the approved `System` taxonomy.
2. Rebuild the surface shell around:
   - section rail
   - browse list
   - detail pane
3. Convert integrations into a browse/detail split with named canonical actions only.
4. Convert control and preferences into bounded row/detail surfaces rather than old panel buckets.
5. Update route-target helpers and focused tests to the new taxonomy.

## Verification

- targeted `System`, neighboring `Now`, `Threads`, and shell tests
- production web build
