# Phase 55 Plan 01

## Goal

Remove superseded shell/detail paths and legacy client-only compatibility seams so the web UI keeps one honest MVP route model before milestone closeout.

## Scope

- shrink the operator surface registry to the still-supported shell routes
- remove placeholder main-panel handling for dead routes
- drop legacy `SettingsPage` tab compatibility that no active caller uses
- remove dead `Suggestions` and `Stats` view files from the web client tree
- update the local source-map docs/tests so they describe the actual shell

## Exit Criteria

- `Now`, `Inbox`, `Threads`, `Projects`, and `Settings` are the only remaining route-level web surfaces in the shell code
- `MainPanel` has no dead placeholder lane for removed surfaces
- `SettingsPage` accepts only the current settings-tab contract
- stale web-only view files for removed lanes are deleted
- focused shell/settings/projects tests still pass after cleanup
