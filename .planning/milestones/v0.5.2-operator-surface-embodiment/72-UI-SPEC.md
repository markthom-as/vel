## Layout Contract

- Desktop posture: split-pane shell. Keep primary surface content dominant; use side panes for secondary inspection, never as equal-weight columns.
- Narrow posture: stack detail beneath or over the primary surface. Do not preserve desktop side-by-side compression below tablet width.
- Shell rhythm: persistent top nav, one active surface at a time, and no global right-side info rail; secondary inspection appears only as per-surface drawer or contextual expansion.
- Visual direction: dark instrument-panel tone. This phase may reset styling strongly, but must not read like a marketing page or launch splash.
- Surface count: exactly `Now`, `Threads`, and `System`. No fourth top-level destination, no resurrected `Inbox` or `Settings`.

## Surface Contract

### `Now`

- Fixed priority gradient order: `Focus`, `Commitments`, `Calendar`, `Triage`.
- `Focus` is singular and visually dominant: one primary focus block, larger than any other section, pinned above the fold on desktop and first in the stack on narrow screens.
- `Commitments` are adjacent canonical work items, not merged into `Focus`.
- `Calendar` is today-first. Show current and near-horizon items only; do not widen into week-planner posture.
- `Triage` is fast and compact. It may expose canonical direct actions, but must remain visually subordinate to `Focus`.
- Capture remains always available, low-friction, and persistent, but visually secondary to the active operating state.
- No material `Now` layout drift is allowed after implementation starts without explicit approval.

### `Threads`

- Primary emphasis is bound object state, not message chronology.
- Thread list should privilege grounded objects, status, and next action over transcript volume.
- Provenance defaults to compressed inline signal with explicit expansion for full detail.
- Chronological message flow remains available, but visually second-order to object context and actionable state.
- Invocation and tool affordances must appear bounded, explicit, and eligibility-based.

### `System`

- Remains one top-level route with internal section switching only.
- Section groups are fixed and visible: `Domain`, `Capabilities`, `Configuration`.
- Desktop uses split-pane browse/detail posture.
- Narrow screens use stacked drill-in posture with a clear back path.
- This surface is read-first. Only named canonical actions may receive accent treatment or primary buttons.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline metadata separation, provenance chips |
| sm | 8px | Compact row spacing, dense control grouping |
| md | 16px | Default element spacing, card padding, form spacing |
| lg | 24px | Section padding, split-pane interior spacing |
| xl | 32px | Major layout gaps, section transitions |
| 2xl | 48px | Large surface breaks, dominant `Focus` breathing room |
| 3xl | 64px | Page-level top/bottom spacing on large desktop surfaces |

Exceptions: `1px` hairline dividers are allowed for pane separation and meter rules; interactive controls must keep a minimum `44px` hit area.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 | 1.5 |
| Label | 14px | 600 | 1.4 |
| Heading | 20px | 600 | 1.2 |
| Display | 28px | 600 | 1.15 |

Additional rules:

- Use only weights `400` and `600`.
- Use mono metadata selectively for timestamps, IDs, provenance tokens, and capability state labels.
- Avoid decorative oversized hero type. `Display` is for the active surface title or singular focus heading only.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#09090B` | App background, primary canvas, inactive shell fields |
| Secondary (30%) | `#18181B` | Cards, nav, side panes, inset sections, grouped trays |
| Accent (10%) | `#FF6B00` | Singular focus treatment, active nav state, canonical primary action, selected object/thread state, live progress emphasis |
| Destructive | `#DC2626` | Disconnect/remove/destructive confirmations only |

Accent reserved for: singular `Focus` emphasis, current active surface indicator, one primary action per panel, selected thread/object row, active progress/risk signal that requires operator attention. Do not use accent for every clickable control.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | Complete commitment |
| Empty state heading | No current operator state |
| Empty state body | Capture a note, sync a canonical source, or return after the next backend update. Do not fabricate placeholder truth. |
| Error state | Unable to load canonical state. Retry this surface; if the problem persists, inspect `System` for connection or capability status. |
| Destructive confirmation | Disconnect Google Calendar: `Disconnect Google Calendar? Calendar sync will stop until you reconnect.`; Disconnect Todoist: `Disconnect Todoist? Task sync will stop until you reconnect.` |

## Interaction and State Rules

- Loading must preserve layout shape with stable skeletons. Do not replace the whole surface with jumping spinners once structure is known.
- Mutations should acknowledge immediately in-place, then reconcile with backend truth. Avoid full-surface refetch flashes after routine actions.
- Keep panel expansion and disclosure predictable. No surprise drawer openings on hover or background refresh.
- Selected state must persist visually through refresh and incremental data updates.
- Motion should be minimal and functional: `150-200ms` ease for pane open/close and emphasis transitions; no decorative parallax, float, or marketing-style reveal choreography.
- Keyboard focus styles must be explicit and high-contrast, using accent plus outline contrast rather than color-only changes.

## Performance Contract

- Surface loads must render a recognizable shell immediately.
- Preserve predictive stability: no major section jumps after actions complete.
- Route switches should reuse shell chrome and avoid blank intermediate frames.
- Refresh only the changed region when possible; do not invalidate the entire surface for a single row mutation.
- Treat visible slowness in `Now` and selection lag in `Threads` as contract failures, not polish debt.
