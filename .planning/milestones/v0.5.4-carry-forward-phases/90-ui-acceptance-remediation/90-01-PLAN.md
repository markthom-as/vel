# Phase 90 Plan

## Objective

Bring the web UI back to an actually passable state against the locked design packet and user acceptance.

## Scope

### In scope

- identify the highest-signal visual/layout/UX failures on the shipped `Now`, `Threads`, and `System` surfaces
- compare current runtime against the locked `0.5.3` spec and prototype packet
- implement remediation in bounded slices
- update proof and acceptance docs only after the UI is genuinely improved

### Out of scope

- pretending proof/build status equals visual acceptance
- broad product redesign outside the locked `0.5.3` packet

## Immediate Next Step

1. Audit the current runtime surface quality against:
   - locked page specs
   - prototype packet
   - user acceptance signal
2. Turn the audit into concrete remediation slices before attempting milestone closeout again.

## Execution Progress

### Completed remediation slices

- `Now`
  - removed the bottom global action bar from the live app shell
  - restored the accepted `Now` header structure:
    - `CLIENT_NAME | LOCATION`
    - `Now`
    - date/time with timezone
    - `current event | context`
  - moved trust out of the main `Now` surface
  - restored right-side header badges for alerts and sync state
  - rebuilt active-task treatment around large left-side completion checkboxes
  - replaced split event/task structure with a mixed `Next up` list
  - replaced the removed event section with `If time allows`
  - moved completed tasks to a collapsed section at the bottom with reopen affordance

- `Nudges`
  - removed encapsulated-card treatment
  - normalized all nudges to the floating slim archived-style layout
  - renamed the surface heading to `Nudges`
  - removed old explanatory helper copy
  - sorted visible nudges by urgency and collapsed overflow behind `+N more`

- `Threads`
  - removed helper text and the rejected `Shared review panel` / `Sticky` side surfaces
  - flattened the layout into a compact rail plus floating thread header plus message stream
  - added collapsible thread rail behavior
  - updated thread-list rows to show relative freshness plus timestamps
  - updated opened-thread header to show title, dates, message count, tags, and archive action

- `System`
  - removed the rejected top helper/hero shell from the surface
  - replaced the old section card plus browse/detail panel stack with:
    - compact two-level left rail
    - contextual `On this page` anchor list
    - full-width scrollable document for the active section
  - kept section selection in the rail while eliminating the separate second navigation panel
  - preserved inline preference editing and immediate toggle behavior
  - retained named integration actions while reducing shell helper noise
