# Phase 102 Plan

## Objective

Implement the Rust engine seam that owns frame movement, turn detection, orchestration, and cancellation.

## Deliverables

- [ ] ring-buffer ingress/egress topology
- [ ] frame/segment/turn domain types
- [ ] resampling and normalization path
- [ ] VAD / turn manager
- [ ] STT/TTS/LLM trait adapters
- [ ] cancellation and flush semantics
- [ ] engine telemetry for latency and underruns

## Execution Checklist

1. Create callback-safe frame ingress contracts.
2. Add engine workers and bounded control channels.
3. Add turn-state machine with interruption/cancel states.
4. Wire STT/TTS/LLM abstractions behind traits.
5. Add engine-level tests for cancellation and single-active-turn truth.

## Exit Criteria

- the engine can accept frames and produce assistant output without platform-specific assumptions
- interruption is a first-class tested path
- no privileged Apple session logic leaks into the engine
