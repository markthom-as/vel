Exercise the Rust engine through a proving adapter and bind it into truthful thread call mode.

## Deliverables

- [ ] proving adapter or harness for PCM ingress/egress
- [ ] thread call-mode integration over the normal assistant path

- [ ] manual proving workflow for a non-Apple environment

## Execution Checklist

1. Implement or wire the proving adapter.
2. Connect call mode to thread state instead of a separate shadow conversation.
3. Add interruption-triggered cancel/flush behavior.
4. Record metrics for latency, drops, and interruption reaction.
5. Run repeated overlap/barge-in manual tests.

## Exit Criteria

- the engine is proven end to end before Apple integration
- call mode does not fork conversation truth away from threads
- the evidence packet shows working interruption and cancel behavior
