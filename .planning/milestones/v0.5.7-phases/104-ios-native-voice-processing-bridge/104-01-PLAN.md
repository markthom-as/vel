# Phase 104 Plan

## Objective

Add the iOS-native duplex path with Apple-managed session/voice-processing behavior feeding the Rust engine.

## Deliverables

- [ ] Swift/Obj-C audio shell for session setup and device events
- [ ] PCM bridge into Rust
- [ ] playback bridge out of Rust
- [ ] interruption and route-change mapping
- [ ] real-device call-mode proof

## Execution Checklist

1. Define the FFI boundary for frames, playback, and device events.
2. Implement session and permission ownership in native code.
3. Feed bridged PCM into the existing Rust engine.
4. Verify interruption, route-change, and speaker/Bluetooth transitions.
5. Run real-device duplex proof and record evidence.

## Exit Criteria

- the iOS path uses native session ownership rather than Rust-side approximation
- the Rust engine remains the single source of truth for turn behavior
- real-device proof exists for the major Apple-specific event classes
