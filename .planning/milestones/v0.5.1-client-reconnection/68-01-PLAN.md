# 68-01 Plan

## Objective

Replace route-by-route fetch behavior with a canonical client transport layer.

## Work

1. Define typed query/mutation boundaries aligned to canonical backend routes.
2. Route all writes through `WriteIntent`.
3. Eliminate direct fetch usage outside the transport layer.

## Acceptance

- one typed query layer exists
- one typed mutation layer exists
- no direct fetches remain outside the canonical boundary
- no local normalization of backend truth survives in client code paths using the new layer
