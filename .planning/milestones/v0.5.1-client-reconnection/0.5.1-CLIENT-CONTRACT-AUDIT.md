# `0.5.1` Client Contract Audit

## Purpose

This Phase 67 audit inventories the currently live web-client touchpoints that still reflect pre-`0.5` surface structure, mixed transport families, or legacy compatibility assumptions.

Every seam listed here is classified as `rewrite`, `quarantine`, or `delete`. No unnamed temporary shim is allowed.

## Audit Summary

The current web client still carries three major pre-`0.5.1` drift classes:

1. legacy first-class surfaces (`Inbox`, `Settings`, hidden `Projects`) that contradict the frozen `Now` / `Threads` / `System` model
2. a mixed transport lane where `operator.ts` combines canonical `v1` reads with quarantined `/api/*` configuration and integration routes
3. direct-fetch escapes inside `SettingsPage.tsx` that bypass the canonical transport boundary altogether

## Named Quarantines

These are the Phase 67 quarantines that are intentionally not reopened during `v0.5.1` implementation:

- `/v1/sync/*` until a canonical `refresh` action is explicit and unambiguous
- `/v1/agent/inspect` until `/system` proves it truly needs this visibility in the fixed section set

## Exit Condition

Phase 67 is only complete if later implementation can treat this audit as law:

- every legacy surface seam has a named disposition
- every stale route family has a named disposition
- direct fetch escapes are forbidden moving forward
- no Phase 68-71 work needs to rediscover whether a seam should be rewritten, quarantined, or deleted
