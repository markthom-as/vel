# 71-01 Plan

## Objective

Delete or quarantine stale seams, prove the truthful web line, and publish the Apple handoff packet.

## Work

1. Apply the named kill list and quarantine only what cannot yet be removed.
2. Delete server endpoints in the same milestone wherever no live callers remain; anything left must be a named quarantine with rationale.
3. Run the must-pass proof set for `Now`, `Threads`, `System`, canonical write flows, dry-run, explain, drift failure, stale-data posture, and no-silent-fallback.
4. Use a lightweight checked-in browser harness with deterministic one-command flows; preserve one script or command and one markdown evidence note per major browser proof.
5. Use controlled fixtures for degraded-state proof and demonstrate no-silent-fallback in the browser rather than only in component or unit tests.
6. Capture evidence at the correct layer for each claim: artifacts for shipped changes, logs plus canonical state for mutations, and browser proof for operator-visible behavior.
7. Document `/v1/agent/inspect` and `/api/integrations/connections` as intentional bounded reads rather than reopening backend read surfaces during closeout.
8. Publish Apple handoff/spec docs only at the behavior/contract level, in both the milestone packet and a durable docs pointer.

## Acceptance

- cleanup success criteria are met
- web proof set passes
- browser proof artifacts include both automated execution evidence and human-readable notes
- degraded-state proof uses controlled fixtures and no-silent-fallback is browser-visible
- every major claim is proven at the correct layer rather than substituted with weaker evidence
- Apple implementation remains out of scope and is replaced by explicit handoff docs
