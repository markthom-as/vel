# 27-03 Plan

## Slice

Use normalized scheduler facets in reflow, recall, and grounded assistant context instead of ad hoc raw-label parsing.

## Why this slice third

This is the product payoff. Phase 27 is only real once the canonical semantics become the shared behavior substrate rather than passive stored metadata.

## Outcomes

- reflow consumes durable scheduler facets first
- recall and assistant grounding can inspect scheduler semantics directly
- agent-facing context becomes more explainable and less dependent on provider syntax leakage
- any remaining fallback parsing is bounded, documented, and secondary

## Likely Touch Points

- `crates/veld/src/services/reflow.rs`
- `crates/veld/src/services/retrieval.rs`
- `crates/veld/src/services/chat/`
- relevant integration tests under `crates/veld/tests/`

## Verification

- targeted reflow tests proving normalized facets drive proposals
- retrieval/grounding tests proving scheduler semantics appear in bounded assistant context
- regression checks that no client shell becomes the owner of scheduler interpretation
