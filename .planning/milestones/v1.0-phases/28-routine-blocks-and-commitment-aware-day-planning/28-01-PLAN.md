# 28-01 Plan

## Slice

Publish the canonical routine-block and day-plan contract over commitments, calendar anchors, and scheduler rules.

## Why this slice first

Phase 27 locked scheduler semantics, but there is still no explicit contract for proactive day shaping. This slice defines the typed planning vocabulary before implementation widens.

## Outcomes

- define typed day-plan outputs such as `scheduled`, `deferred`, `did_not_fit`, and `needs_judgment`
- define how routine blocks participate as backend-owned planning inputs
- document the relationship between initial day planning and later `reflow`
- ship checked-in examples or templates if the boundary becomes stable enough

## Likely Touch Points

- `crates/vel-core/src/`
- `crates/vel-api-types/src/`
- `docs/cognitive-agent-architecture/architecture/`
- `docs/product/`
- `config/examples/` and `config/schemas/` if the contract stabilizes in this slice

## Verification

- focused Rust tests for typed planning-contract/domain behavior
- fixture/example parse checks if schema-bearing assets are added
- `rg` checks that docs teach backend-owned planning semantics rather than shell heuristics
