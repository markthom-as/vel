<objective>
SP2 Lane C — Capability Broker MVP (ticket 016). Introduce a brokered capability boundary so agent requests are mediated by explicit scope checks, secrets are injected only inside the broker execution path, and all grant/deny/execution decisions are persisted with stable trace IDs.

Purpose: CAP-01 (agents request scoped permissions, not raw credentials) and CAP-02 (prompt-visible raw credentials prohibited) close the most critical security gap in the current system. Without this seam, Phase 3 agent evaluation work would run on an unsafe foundation.

Output: `CapabilityDescriptor`, `CapabilityGrant`, `CapabilityDenial` in vel-core. `BrokerService` in veld/services. `broker_events` migration and repository. All approvals and denials emitted with traceable event IDs.
</objective>

    BrokerService behavior:
    - `fn resolve_capability(run_id, requested_scope, allowlist: &[CapabilityDescriptor]) -> Result<CapabilityGrant, CapabilityDenial>`
      - If requested_scope matches any descriptor in allowlist → return CapabilityGrant
      - If no match → return CapabilityDenial with reason "scope not in allowlist"
      - Emit and persist broker event regardless of outcome
    - `fn deny_with_trace(run_id, descriptor, reason) -> CapabilityDenial`
      - Convenience: always-deny path for fail-closed behavior
      - Persists denial event
    - Secret injection rule: if a capability grants access to a provider (e.g. "execute:todoist"), the broker resolves the secret from the config/keystore at point-of-use inside the broker and passes a pre-resolved HTTP request or action result — never the raw secret — back to the caller

    - Index: `idx_broker_events_run_id` on `run_id`

       ```rust
       #[derive(Debug, Clone, PartialEq, Eq, Serialize, Deserialize)]
       pub struct CapabilityDescriptor {
           pub scope: String,   // e.g. "read:context"
           pub resource: Option<String>,
           pub action: String,  // e.g. "read"
       }

       impl CapabilityDescriptor {
           pub fn matches(&self, requested: &CapabilityDescriptor) -> bool {
               self.scope == requested.scope && self.action == requested.action
               // resource: None in allowlist descriptor = wildcard (matches any resource)
               && (self.resource.is_none() || self.resource == requested.resource)
           }
       }
       ```
