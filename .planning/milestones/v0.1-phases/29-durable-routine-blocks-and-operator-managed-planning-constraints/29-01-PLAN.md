# 29-01 Plan

## Slice

Publish the durable routine-block and planning-constraint contract over the existing backend-owned day-plan substrate.

## Why this slice first

Before persistence or shell work expands, Vel needs one canonical contract for what a routine block is, what a bounded planning constraint is, and how both feed `day_plan` / `reflow`.

## Outcomes

- canonical domain vocabulary for durable routine blocks and bounded planning constraints
- explicit relationship to existing `day_plan` and `reflow` contracts
- schema/example/owner-doc path identified if the boundary is stable enough
