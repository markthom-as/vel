# Phase 99 Plan

## Objective

Complete the accepted web operator experience on top of the truthful Phase 98 seams.

## Implementation slices

1. Navbar, docs-frame, and composer completion
   Status: complete.
   Scope:
   - ensure sync icon temperament matches real sync activity
   - normalize navbar tag sizing for current event and active task
   - render docs properly within the site frame
   - finish composer shadow, disabled state, and onboarding-blocked presentation

2. Nudges and onboarding flow completion
   Status: complete.
   Scope:
   - finish sticky nudge behavior with navbar
   - finish nudge pill/ring/action-chip geometry

3. `Now` completion and day-operator polish
   Status: complete.
   Scope:

4. `Threads` completion and call-mode entry UX
   Status: complete.
   Scope:
   - finish last-thread vs empty-state handling
   - finish no-tail modern chat layout and readable Vel bubble styling
   - finish archive control standardization

5. `System` operator-mode and developer-mode completion
   Status: complete.
   Scope:
   - finish Core settings structure and readability
   - finish integration/service rows with icons and truthful status tags
   - keep unavailable services visible but collapsed
   - hide MVP-irrelevant technical sections behind developer mode without making the operator path feel incomplete

## Verification

- browser/manual review in desktop Chrome for each touched surface
- targeted frontend tests only where they help guard interaction regressions
- web build after cohesive polish slices

## Success Criteria

1. The MVP operator flow feels coherent from onboarding through daily use without visual or interaction drift.
2. `Now`, `Threads`, and `System` all read as one product surface family rather than separate passes.
3. Phase 100 can focus on proof and honest closeout rather than additional UX discovery.
