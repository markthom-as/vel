# Milestone v0.5: Backend Core Rewrite

**Status:** SHIPPED 2026-03-23
**Phases:** 57-65
**Total Plans:** 36

## Overview

`v0.5` rebuilt Vel around canonical backend authority:

- first-class canonical objects as the system of record
- a typed action membrane for all reads and writes
- policy, grants, ownership, and audit as mandatory infrastructure
- governed module bootstrap and activation
- manual workflow runtime over canonical objects
- native calendar core plus canonical availability semantics
- Todoist and Google Calendar as proving adapters
- hard cutover to canonical write authority

This milestone intentionally excluded new client/UI work and broader connector expansion. It closed through execution-backed proof rather than a long dual-live compatibility lane.

## Accomplishments

- Phase `57` froze the canonical object, relation, membrane, workflow, module, task, and provider-integration contracts
- Phase `58` landed the storage/system-of-record substrate for canonical objects, links, runtime records, projections, and migration/bootstrap seams
- Phase `59` made reads and writes flow through typed action, policy, ownership, audit, and `WriteIntent` infrastructure
- Phase `60` proved governed module loading, deterministic bootstrap, and policy-mediated provider-module registration
- Phase `61` landed a manual, mediated, refusal-aware workflow runtime over canonical objects
- Phase `62` made calendar/event/recurrence/attendee/availability semantics native to the core
- Phase `63` proved Todoist as a canonical task adapter with ownership-aware sync, `TaskEvent` history, tombstones, and mediated writes
- Phase `64` proved Google Calendar as a canonical calendar adapter with recurrence fidelity, availability bridging, tombstones, and mediated writes
- Phase `65` cut over live write authority to canonical routes, quarantined superseded legacy write paths, and closed the milestone with integrated proof
