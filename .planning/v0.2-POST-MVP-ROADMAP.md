# v0.2 Post-MVP Roadmap

`v0.2` shipped the bounded MVP loop:

`overview -> commitments -> reflow -> threads -> review`

This file records what was intentionally deferred so the milestone can close without implied carryover scope.

## Deferred from v0.2

The following work is explicitly out of scope for the shipped MVP:

- multi-day planning, autonomous planning, or planner redesign beyond same-day repair
- broad provider, connector, or platform expansion
- broad shell redesign outside the MVP loop surfaces
- Apple-local planner logic or broad FFI migration
- broad calendar write-back automation
- generic chat or tool-product expansion beyond bounded thread continuation
- heavy analytics, journaling, scoring, or routine-builder redesign

## What v0.2 Proved

`v0.2` closes with these truths in place:

- one strict, verified daily operator loop exists across shipped surfaces
- MVP behavior is backed by Rust-owned contracts, services, and typed transport
- web and Apple shells are thin embodiments over the same backend-owned loop
- reflow remains bounded, explainable, and review-gated
- threads remain the supervised continuation lane for non-trivial follow-through

## Post-MVP Work Lanes

These are the next meaningful lanes after `v0.2`. They are roadmap candidates, not retroactive MVP scope.

### 1. Review Depth And Carry-Forward Closure

Tighten the review lane beyond the compact MVP closeout:

- richer carry-forward handling between end-of-day review and the next day’s overview
- better review navigation across commitments, reflow outcomes, and thread-mediated actions
- bounded historical comparison or trend views only if they stay explainable from persisted runs

### 2. Planning Beyond Same-Day Reflow

Extend planning only after the current-day loop stays stable:

- forward-looking schedule exploration beyond the compact current-day contract
- durable planning-profile refinement and operator-managed constraints
- carefully bounded calendar write-back or supervised proposal application expansion

### 3. Threads And Tool Continuation Expansion

Broaden continuation only if provenance and review posture stay intact:

- deeper context recall and project/data grounding inside threads
- more explicit review/apply flows for assisted actions
- additional bounded tool capabilities that preserve deny-by-default access and typed provenance

### 4. Apple Validation And Surface Hardening

Complete the Apple proof in a real host environment:

- run Swift package checks and app-target validation on a machine with full Apple toolchains
- verify the MVP shell hierarchy and typed transport in real Apple app flows
- harden shell-native wrappers such as quick entry or voice without reintroducing shell-owned product logic

### 5. Broader Platform Roadmap

Only after the MVP loop is stable:

- broader provider/integration expansion
- deeper memory or recall lanes beyond the current bounded MVP needs
- later architecture work already deferred from earlier milestones

## Closeout Rule

Anything not required to keep the shipped loop truthful, Rust-owned, and cross-surface portable belongs after `v0.2`, not inside it.
