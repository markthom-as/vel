# Active Milestone Requirements: `v0.5.2`

## Milestone

`v0.5.2` — Operator Surface Embodiment

## Goal

Turn the truthful `0.5.1` client line into the intended operator UI state on web without reopening backend law.

## Invariant

- backend truth remains frozen except for provable bugs
- client-side semantic invention remains forbidden
- web is the implementation target
- Apple receives parity/handoff documentation only
- no new top-level surfaces are added beyond `Now`, `Threads`, and `System`

## Requirement Buckets

| ID | Description |
|----|-------------|
| DOCTRINE-52-01 | A standalone operator-surface doctrine defines ideal-state UI law for `Now`, `Threads`, and `System`. |
| SHELL-52-01 | Shared shell, layout primitives, disclosure rules, and navigation rhythm support the three-surface model cleanly. |
| NOW-52-01 | `Now` becomes a denser execution-first surface over canonical task/calendar truth without synthetic cross-type ranking. |
| THREADS-52-01 | `Threads` becomes a clearer grounded interaction surface over canonical thread and object truth. |
| SYSTEM-52-01 | `System` becomes a legible structural/configuration surface with stable sections and only named canonical actions. |
| PERF-52-01 | Visible latency and unnecessary churn are reduced in the active web operator path. |
| VERIFY-52-01 | Browser proof, focused tests, build verification, and milestone evidence close the line honestly. |
| APPLE-52-01 | Apple receives updated parity/handoff documentation only. |

## Non-Goals

- backend schema or ontology renegotiation
- framework migration
- new providers
- workflow-builder or trigger-product widening
- Apple implementation
