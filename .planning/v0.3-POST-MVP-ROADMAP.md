# v0.3 Follow-On Roadmap

This file records work that remains after closing `v0.3`.

## Highest-Value Follow-On Work

- full Apple app-target validation in a macOS/Xcode environment to replace the current source-level parity limit with execution-backed app validation
- operator-visible client linking and recovery flows that move beyond compact posture into deeper guided repair where needed
- additional refinement of reduced-watch continuity and handoff once real usage shows where compact follow-through still leaks friction
- broader `Now` polish and density tuning after the canonical cross-platform contract is stable in repeated use

## Explicitly Not Reopened By Default

- shell-local `Now` semantics
- a second planner lane outside the canonical Rust-owned `Now` and thread model
- generic chat-product widening
- multi-day planning expansion not required by the current-day contract

## Trigger To Reopen

Reopen roadmap work here only when:

- repeated operator use exposes a real continuity/parity gap, or
- Apple app-target validation reveals a concrete mismatch between the shared contract and a shipped shell, or
- client-mesh recovery still causes operators to fail to join the same authority runtime reliably
